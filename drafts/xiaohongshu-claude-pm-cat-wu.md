## ✍️ 正文

家人们，这篇真的建议所有做产品、做内容、做营销的姐妹都存一下📍

Claude Code 的产品经理 Cat Wu 刚发了一篇博客，**把她在 Anthropic 内部怎么用 AI 重构自己 PM 工作流** 讲得明明白白。

看完我只有一个感觉：

> **AI 时代，人人都是产品经理。你不是在和 AI 抢工作，你是在用 AI 重做自己的工作。**

先抛一个让我后背发凉的数据👇

🔹2024 年 10 月：Sonnet 3.5 只能完成人类 21 分钟的软件任务
🔹2026 年 3 月：Opus 4.6 能搞定人类 **12 小时** 的软件任务

**16 个月，41 倍。**

这意味着你今天绕开的"模型做不到"，下个月可能就变成多余的设计💡

---

## 🚨 老一套 PM 为什么不灵了？

过去产品经理的逻辑：

❌ 前期收集信息 → 做自信判断 → 按计划推进几个月

但模型是**指数级**在变强的。
你辛苦设计的"绕路方案"，模型一更新，全成了多余复杂度🫠

Cat Wu 的一句话我直接截图了：

> **"地面在你脚下持续抬升，团队必须围绕这个现实重新组织工作。"**

新节奏只有三个词：
**快速实验 · 持续交付 · 押注有效方向**

---

## 🛠️ 她的 AI 工作流分工（照抄版）

她把工作拆给 3 个工具，非常清晰👇

1️⃣ **Claude.ai** — 动脑
   打磨策略、聊复杂情况、快速问答

2️⃣ **Claude Code** — 动手
   搭原型、跑评估、写脚本（她从没手写过一行代码🤫）

3️⃣ **Cowork** — 动杂事
   清收件箱、做 PPT、翻 Slack 历史、订差旅

▶️ **启发**：别纠结"用哪个 AI"，而是想清楚**每类任务交给谁**。

---

## 💥 她做出的 4 个关键转变（划重点‼️）

### 1. 用「短冲刺」代替「长路线图」

团队每个人都可以开 **Side Quest（边缘探索）**：
一个下午 → 做一个你以为做不到的原型。

Claude Code 桌面版、待办清单功能…… 全是这样被"玩"出来的🎮

### 2. 用「Demo + 评估」代替「文档」

站会不再是汇报，而是**放演示**。
一个小技巧抄走👇

> 写完规范文档后，**把它丢给 Claude Code 让它直接搭**，哪怕粗糙，也能彻底改变团队讨论的质量。

而评估（eval）会把抽象功能变具体——**能度量，才能改进**📊

### 3. 每个新模型出来，**回头重做旧功能**

最好的方式：**自己成为重度用户**，故意去试那些"你以为做不到的事"。

👀 **反直觉金句**：
"原型阶段要优先能力上限，别过早砍 token 成本。"
先验证能不能做，成本让下一个更便宜的模型来扛。

### 4. 做最简单、能跑通的方案

一开始 Claude Code 的待办列表要靠"系统提醒"硬推模型去勾选✅
下一个模型出来——**这个能力直接内置了，补丁全删**。

> 越是巧妙的绕路，下一个模型出来就越是累赘。
